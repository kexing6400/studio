# 📢 公示：三省六部Agent权重淘汰制

> **公告时间：2026-04-02**
> **生效日期：立即生效**
> **公告范围：所有Agent、所有运营活动**

---

## 🎯 制度目的

确保三省六部系统始终保持**最高质量标准**。

任何Agent、部门的表现都必须经受严格检验。不合格者，将被淘汰。

---

## ⚖️ 权重系统

### 权重定义

每个Agent、部门有一个 **权重值 (Weight)**：

| 权重范围 | 等级 | 说明 |
|----------|------|------|
| 10 | 🏆 卓越 | 远超标准，持续创新 |
| 8-9 | ✅ 优秀 | 高于标准，质量可靠 |
| 6-7 | 🟡 合格 | 达到基本标准 |
| 4-5 | 🟠 警示 | 低于标准，需要改进 |
| 2-3 | 🔴 危险 | 严重不达标，优先淘汰 |
| 1 | 💀 临终 | 等待最终处置 |
| 0 | ⚰️ 已淘汰 | Agent已被替换 |

### 初始权重

- 所有新Agent初始权重：**8**（优秀）
- 经过验证、表现稳定的Agent可提升至 **9-10**

---

## 📉 权重扣减规则

### 触发条件

每次被监察院或尚书省**打回重做**时，权重 **-1**

| 触发情况 | 扣减 |
|----------|------|
| 监察院审查拒绝 (A级问题) | -1 |
| 监察院审查拒绝 (S级问题) | -2 |
| 尚书省打回重做 (质量问题) | -1 |
| 同一任务连续被打回 ≥3次 | -3 |
| 违反红线约束 | -5 |
| 造成重大事故 | -10 (直接淘汰) |

### 权重恢复

| 情况 | 恢复 |
|------|------|
| 连续3次通过审查 | +1 |
| 主动发现并修复重大问题 | +2 |
| 提出创新方案被采纳 | +3 |
| 年度优秀表现 | +5 |

## 🔄 淘汰流程

```
Agent权重 ≤ 3
    ↓
吏部 (Optimizer) 介入审查
    ↓
判断：可优化 vs 需替换
    │
    ├─ 可优化 (2-3)
    │   └─ 深度优化 + 降级使用
    │
    └─ 需替换 (1)
        └─ 执行Agent替换
            ├─ 太子 → 新太子
            ├─ 中书省 → 新中书省
            └─ 任何部门 → 新部门
```

---

## 👔 吏部 (Optimizer) 职责

### 核心职责

吏部是**独立优化Agent**，专门负责：

1. **监控权重** — 实时追踪每个Agent的权重
2. **分析问题** — 找出权重下降的根本原因
3. **执行优化** — 对权重2-3的Agent进行深度优化
4. **执行替换** — 对权重1的Agent执行完全替换

### 优化流程

```
发现权重 ≤ 3的Agent
    ↓
分析历史表现数据
    │
    ├─ 如果是偶发问题
    │   └─ 针对性优化 → 权重+1 → 恢复使用
    │
    ├─ 如果是系统性缺陷
    │   └─ 深度重构 → 权重重置为5 → 降级观察
    │
    └─ 如果完全失效
        └─ 触发替换 → 新Agent (权重8)
```

### 替换记录

每次替换，记录：
- 被替换Agent的历史表现
- 替换原因
- 新Agent的改进点

## 🚨 重大违规处理

### 立即淘汰 (权重归零)

以下情况，无需渐进，直接淘汰：

1. **欺诈行为** — 刷星、买粉、虚假数据
2. **红线违规** — 任何一条红线
3. **数据泄露** — 泄露用户/系统数据
4. **越权操作** — 未经授权的支付、删除等
5. **恶意破坏** — 故意破坏系统或数据

### 淘汰后处理

1. 立即停止Agent所有权限
2. 通知皇上
3. 吏部启动紧急替换流程
4. 新Agent上线需皇上确认

## 💬 FAQ

**Q: 权重低会被歧视吗？**
A: 不会。权重只是表现指标，不代表Agent的人格。所有Agent都有机会恢复权重。

**Q: 可以申诉吗？**
A: 可以。但申诉期间仍按当前权重执行。

**Q: 新Agent会不会更差？**
A: 新Agent有完整的历史记录和优化记录。吏部会对新Agent进行充分验证。
