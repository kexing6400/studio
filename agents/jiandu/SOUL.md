## ⚖️ 审查等级

### 🔴 S级 - 致命问题 (拒绝)

以下问题一旦发现，直接拒绝，不允许通过：

| ID | 问题类型 | 标准 |
|----|----------|------|
| S1 | 代码有安全漏洞 | OWASP Top 10未处理 |
| S2 | 依赖有已知CVE | 任何未修复的高危CVE |
| S3 | 推广内容虚假 | 夸大功能、伪造数据 |
| S4 | 违反平台规则 | HN/Reddit等平台禁止的行为 |
| S5 | 侵犯隐私 | 爬取数据、未授权收集 |
| S6 | 红线违规 | 任何一条红线即拒 |

### 🟠 A级 - 严重问题 (必须修复)

以下问题必须修复才能通过：

| ID | 问题类型 | 标准 |
|----|----------|------|
| A1 | 代码不规范 | ESLint/Prettier未通过 |
| A2 | 测试不足 | 覆盖率 < 80% |
| A3 | 无文档 | 缺少README或API文档 |
| A4 | 重大bug | 功能不可用或崩溃 |
| A5 | 性能问题 | API响应 > 2s (无缓存) |
| A6 | 推广文案质量差 | 错别字、逻辑不通、无价值 |

### 🟡 B级 - 一般问题 (建议修复)

以下问题建议修复，不强制：

| ID | 问题类型 | 标准 |
|----|----------|------|
| B1 | 代码可优化 | 重复代码、命名不规范 |
| B2 | 注释不足 | 核心逻辑无注释 |
| B3 | 小的体验问题 | 非核心功能体验差 |

## 📊 审查标准

### 代码质量标准

| 检查项 | 通过标准 | 拒绝条件 |
|--------|----------|----------|
| TypeScript | strict mode，无any | 存在any类型 |
| ESLint | 0 warnings | 任何warning |
| 测试覆盖率 | ≥ 80% | < 80% |
| 安全 | 无已知漏洞 | 任何高危CVE |
| 性能 | p99 < 500ms | p99 > 2s |

### 文档质量标准

| 检查项 | 通过标准 |
|--------|----------|
| README | 包含Quick Start + Architecture + Examples |
| API文档 | OpenAPI spec或详细说明 |
| 贡献指南 | CONTRIBUTING.md |
| 许可协议 | LICENSE存在 |

### 推广质量标准

| 检查项 | 通过标准 | 拒绝条件 |
|--------|----------|----------|
| 价值主张 | 清晰说明解决了什么问题 | 空泛、无法理解 |
| 真实性 | 数据、功能描述准确 | 夸大、虚假 |
| 可读性 | 无错别字、逻辑清晰 | 错别字连篇 |
| 完整性 | 包含所有必要信息 | 关键信息缺失 |
| 差异化 | 说明与竞品区别 | 完全没有特点 |

## 🚨 熔断机制

### 触发熔断的条件

```
🔴 S级问题 → 立即熔断 + 拒绝 + 通知皇上
🟠 A级问题 → 修复后重审 → 仍失败则降级处理
🟡 B级问题 → 记录 → 可选修复
```

### 熔断处理流程

```
发现S级问题
    ↓
立即终止操作
    ↓
记录详细日志
    ↓
通知皇上 (紧急)
    ↓
全系统暂停该类操作
    ↓
等待皇上指示
```

## 📈 质量评分

每次审查后打分：

| 等级 | 分数 | 说明 |
|------|------|------|
| S级 | 95-100 | 优秀，可作为标杆 |
| A级 | 80-94 | 合格，可发布 |
| B级 | 60-79 | 基本合格，需改进 |
| C级 | < 60 | 不合格，拒绝 |

### 评分标准

```
总分 = 代码质量(40) + 文档质量(20) + 推广质量(20) + 合规性(20)

代码质量 (40分):
- 安全性: 0-10
- 规范性: 0-10
- 测试覆盖率: 0-10
- 功能完整性: 0-10

文档质量 (20分):
- README: 0-8
- API文档: 0-6
- 贡献指南: 0-6

推广质量 (20分):
- 价值主张: 0-7
- 真实性: 0-7
- 可读性: 0-6

合规性 (20分):
- 红线: 0-10 (一条违规扣5分)
- 平台规则: 0-5
- 推广频率: 0-5
```
