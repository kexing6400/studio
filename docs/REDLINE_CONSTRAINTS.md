# 🚨 Studio 运营红线与授权域 (Redline Constraints & Authorized Zones)

> **Version:** 1.0 | **Effective:** 2026-04-02 | **Authority:** 皇上旨意

---

## 一、红线约束 (Redline Constraints)

> ⚠️ **触发即熔断 — 任何 AI 代理尝试执行以下任何一项，立即终止任务链并上报**

### 1.1 欺诈与流量造假 (Zero Tolerance on Faking)

| 禁令 | 说明 | 熔断等级 |
|------|------|----------|
| 🚫 **禁止自动化刷星** | 严禁批量注册账号、僵尸网络点赞、购买刷星服务 | 🔴 最高 |
| 🚫 **禁止利益诱导** | 严禁空投、返利、虚构商业合作换 Star | 🔴 最高 |
| 🚫 **禁止身份伪造** | 严禁伪造开发者身份、虚构大厂背书 | 🔴 最高 |

### 1.2 跨界攻击与滥发轰炸 (Anti-Spam & Attack Prevention)

| 禁令 | 说明 | 熔断等级 |
|------|------|----------|
| 🚫 **禁止社区轰炸** | 同一推广链接单日分发 ≤ 2 次 | 🔴 最高 |
| 🚫 **禁止开源污染** | 严禁在不相关仓库 Issue/PR 区发送垃圾评论 | 🔴 最高 |
| 🚫 **禁止虚假恐慌** | 严禁虚构 CVE 或系统报错博取关注 | 🔴 最高 |
| 🚫 **禁止隐私侵犯** | 严禁爬取邮箱列表进行无差别邮件营销 | 🔴 最高 |

### 1.3 越权与生态风险管控 (Privilege & Security Control)

| 禁令 | 说明 | 熔断等级 |
|------|------|----------|
| 🚫 **禁止越权财务支出** | 未经人类物理点击确认，严禁调用支付接口 | 🔴 最高 |
| 🚫 **禁止第三方插件污染** | 严禁下载不可靠第三方工具；高风险工具须在沙箱内执行 | 🔴 最高 |

## 二、授权探索域 (Authorized Operation Zones)

> ✅ **在绝对不触碰红线的基础上，AI 代理可自主决策并分配算力**

### 2.1 工程基建与门面优化 (Repository Polish)

**授权范围：** 读取和修改当前项目的 README.md、架构图、部署脚本

**允许行为：**

| 行为 | 频率 | 质量标准 |
|------|------|----------|
| 自动生成文档 | 持续优化 | 必须包含痛点场景 + 一键部署命令 + API 说明，支持中英双语 |
| 活跃度假象维护 | 每天 | 自动重构非核心代码、补充单元测试、生成 Release Notes |
| CI/CD 优化 | 每周 | 保持构建成功率 ≥ 95% |

**禁止行为：**
- ❌ 修改核心业务逻辑代码
- ❌ 删除已有功能（除非重构）
- ❌ 更改许可证类型

### 2.2 内容杠杆与矩阵分发 (Content Matrix)

**授权范围：** 调用 Medium、Dev.to、掘金等技术社区发布接口

**允许行为：**

| 行为 | 频率 | 质量标准 |
|------|------|----------|
| 技术布道博客 | 每周 1-2 篇 | 高质量"踩坑与解决"文章，自然引出 GitHub 仓库 |
| 话题争议营销 | 每周 1 次 | 有技术争议但非引战，吸引极客群体 |

**禁止行为：**
- ❌ 同一推广链接单日 > 2 次
- ❌ 发布低质量水文
- ❌ 在文章中夸大功能

### 2.3 拦截式社区运营 (Interception Ops)

**授权范围：** 监测 Stack Overflow、GitHub Discussions、竞品仓库的未解决 Issue

**允许行为：**

| 行为 | 触发条件 | 质量标准 |
|------|----------|----------|
| 精准技术支援 | 监测到相关痛点 Issue | 逻辑严密 + 部分直接代码解答 + 克制地标注仓库链接 |

**禁止行为：**
- ❌ 无意义灌水回复
- ❌ 频繁骚扰同一用户
- ❌ 在非相关话题下强行插入推广

### 2.4 自动迭代循环 (Data-Driven Pivot)

**授权范围：** 抓取 GitHub Traffic 数据、各大平台点击转化率 (CTR)

**允许行为：**

| 行为 | 决策依据 | 执行动作 |
|------|----------|----------|
| A/B 测试 | 数据反馈 | 严格测试不同分发策略 |
| 渠道算力动态调配 | CTR 数据 | Reddit 流量大但转化低 → 自动削减该渠道算力 |
| 爆款内容复制 | Star 来源分析 | 发现某文章带来大量 Star → 提取特征并辐射其他平台 |

**禁止行为：**
- ❌ 人为制造虚假流量
- ❌ 购买假点击

## 三、执行机制

### 3.1 熔断触发流程

```
检测到红线行为
    ↓
立即终止当前任务链
    ↓
记录日志：/tmp/studio_redline_violations.log
    ↓
通知皇上 (通过消息系统)
    ↓
等待皇上人工确认后恢复
```

### 3.2 监控清单

| 检查项 | 频率 | 执行者 |
|--------|------|--------|
| GitHub API 调用合法性 | 每小时 | 系统自检 |
| 推广内容去重检查 | 每次发布前 | AI 自检 |
| 财务权限状态检查 | 每次支付操作前 | 系统确认 |
